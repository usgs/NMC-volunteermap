These are the current instructions for implementing the USGS Visual Identity templates for web applications that are not moving into the USGS Drupal site.

Quick Installation Instructions:

Note: Before implementing these templates, make a backup of the pages in your site that you will be changing.

(1) Prepare Your Website

If you don't already have a directory folder for "images" and "stylesheets", create them at the root of your sub-domain.

If you will be using the "header.txt" and "footer.txt" files as server-side includes, create an "includes" directory in which to place those files.

(2) Download the Appropriate Template

Download either the public template file, or the internal template file, and unzip it to your desktop:

  • Public template (ZIP FILE)
  • Internal, USGS-only website template (ADD ZIP FILE) Coming soon!

Copy the "common.css" file to a domain-level "/stylesheets" directory.

Copy all font files to a domain-level "/fonts" directory.

Copy all image files to a domain-level "/images" directory.

Copy the "custom.css" file to a "/stylesheets" subdirectory of your application.

If you want to use include files, copy the "header.txt" and "footer.txt" files to an "includes" directory you set up and then customize both files for your site .

(3) Update Your Web Pages

There are example HTML files included in each .zip file. They show a complete page demonstrating the proper order for referencing the CSS, illustrate how the template header and footer code should look, and provide all the basic HTML needed to build a page.

Updating Your Pages:

Links to the CSS files are included in the header.txt file; both the CSS files must be linked in this order for the pages to display properly:

Update the document title ( Untitled Document) in the "header.txt" file and add your existing CSS styles to the "custom.css" file. Then replace your page's existing header code with the new header.

Replace your page's existing footer code with the "footer.txt" code.

Congratulations! You're done!

General Notes on Customizaton:

You can hard-code your information on each HTML page, or you can use a server-side include to fill this information in automatically, depending on the type of web server you're using.

You can add your own custom styles to the "custom.css" file, but you MUST NOT add any styles that will modify any part of the USGS header and footer look and feel, or functionality in any fashion other than what is allowed to be modified.

The header and Footer are designed to accomodate visual and/or data intensive applications that utilize every bit of the browser's display window. Because of this, by default, THERE IS NO MAX WIDTH. If you want the want to restrict the width of the header and footer, instructions and code are included in the custom.css file.

Customizing the Header

DO NOT ADD links, logos, or any other items within the header bar—this is strictly prohibited.

Logo Modification

The default identifier height is 65 pixels. Instructions and code are included in the custom.css file to modify the height of the identifier.

  • The height must be at least 50 pixels.
  • Do not alter the aspect ratio of the logo.

Search

By default, the search box is displayed except on the smallest of devices (under 500 pixel width). Instructions and code for hiding the search box are included in the custom.css file. You may replace the search current search function with your own.

Standard Content

By default, USGS has set the font size, family, etc. in order to provide a consistent size for content across all pages. If you would prefer not to use these pre-defined fonts/styles for the main content of your page, you can specify your own. However, you are prohibited from changing the fonts/styles for the header and footer areas under any circumstances. They are included in the custom.css file.

Customizing the Footer

All of the text links in the footer are required.

You may hide the main social media links. Instructions and code are included in the custom.css file.

Google Tag Manager

GTM code included in the header.txt and footer.txt is required and must be included in all pages.
